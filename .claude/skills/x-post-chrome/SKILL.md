# X投稿（Chrome経由）

## 前提チェック（最初に必ず）
1. ブラウザ操作ツール（タブを開く・クリック・入力）が使えるか確認する。
   使えない場合は作業を中止し、ユーザーに伝える:
   「この手順はデスクトップ版Claude Code＋Chrome拡張の接続が必要です。クラウド（Web版）セッションでは実行できません」
2. `git pull` で最新化し、`data/x-posted-log.json`（投稿済みログ）を読み込む。

## 手順

### 1. ネタ集め（トレンド活用）
- Chromeで `https://x.com/explore/tabs/trending` を開き、日本のトレンドを確認する。
- あわせてホームタイムラインの「おすすめ」に出やすい話題（新作カプセルトイ、キャラもの、季節ネタ）も確認する。
- ガチャガチャ・カプセルトイに自然に絡められるトピックを1つ選ぶ。
  無理に絡めると不自然になるので、合うものが無ければサイトの実データ
  （data/spots.js の店舗・地域ネタ）を使ってよい。

### 2. 文面作成（ルール厳守）
- **本文（URL除く）は50文字以内**。文字数を数えて確認する。
- **改行で段落を分ける**（1行目=フック、2行目=本文、3行目=URL任意）。
- **過去投稿と重複しない**: data/x-posted-log.json の全 text と照合する
  （URL・空白を除いた本文が一致したらNG。似すぎもNG）。
- トーンは自然体。絵文字は1〜2個。ハッシュタグは付けても1個まで。
- リンクを入れる場合は https://gacha-hiroba.com/ 配下の実在ページのみ。

### 3. 投稿
- Chromeで `https://x.com/compose/post` を開く（またはホームの投稿ボタン）。
- 文面を入力し、**投稿前にスクリーンショット等で本文を目視確認**してから投稿する。
- 投稿できたことを確認する（タイムラインに反映されたか）。

### 4. ログ共有（重要）
- 投稿した文面を `data/x-posted-log.json` の posts 配列に追記する:
  `{ "date": "<ISO日時>", "slot": "chrome", "text": "<投稿文>" }`
- commit して push する（これでAPI自動投稿側も同じ文面を再投稿しなくなる）。
  コミットメッセージ例: `x-post: chrome投稿ログ追加 [skip ci]`

## 運用メモ
- 定時（12:00 / 16:00 JST）はGitHub ActionsのAPI投稿が自動で走っている。
  Chrome投稿はそれに**追加**する形。同じ日に同じネタを重ねない。
- APIを止めたい場合はユーザーの指示を受けてから
  `.github/workflows/x-post.yml` の schedule を無効化すること（勝手にやらない）。
